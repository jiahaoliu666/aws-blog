解決當輸入密碼並點擊確認刪除的按鈕後出現log輸出遇到的問題，log輸出如圖，需求如下

1.正常要能刪除對應用戶儲存在aws-blog-feedback、aws-blog-avatar的S3檔案
，關於儲存到S3的哪個地方，這部分可以參考以下檔案
src/hooks/profile/useProfileAvatar.ts
src/hooks/profile/useProfileFeedback.ts

2.刪除資料庫數據是要刪除以下資料表名稱當中userId為主鍵的數據
'AWS_Blog_LineVerifications',
'AWS_Blog_UserActivityLog',
'AWS_Blog_UserFavorites',
'AWS_Blog_UserNotifications',
'AWS_Blog_UserNotificationSettings',
'AWS_Blog_UserPreferences',
'AWS_Blog_UserProfiles',
'AWS_Blog_UserRecentArticles'

3.要根據cognito的sub屬性當中的值來刪除對應的cognito用戶 (這值是取src/context/AuthContext.tsx當中的sub值)

4.確保每個步驟都成功才刪除，否則回滾

5.當點擊確認刪除按鈕要顯示對應的toast通知消息，例如：密碼輸入是否正確，4xx、5xx等狀態錯誤或伺服器問題等等

6.都要使用 AWS SDK for JavaScript v3 完成的，所以要使用 AWS SDK v3 的最新 API，這部分可以參考以下官方文檔
https://docs.aws.amazon.com/AWSJavaScriptSDK/v3/latest/

7.每個步驟都要有詳細的log，這樣才知道從哪邊調試

8.刪除的執行流程要人性化，且流程要正常依照順序，不要有成功消息 但執行失敗的問題，反之

9.要能正確匹配驗證cognito用戶的密碼才能才能執行刪除流程，否則顯示對應的toast通知消息和不執行刪除流程

目前另外有個問題就是當密碼輸入錯誤還會執行刪除流程，這部分要仔細確認

以上修改不要影響到現行功能
